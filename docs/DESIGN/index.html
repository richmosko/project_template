10.1 FAQs

Q: How do I add review comments to this design doc?
A: Same as the other docs. Hand-edit docs/DESIGN/comments.md with ## §<section-id> blocks pointing at a section's id (e.g. tokens, flows), or run /serve-docs DESIGN and use the inline + Comment widget. Then run /refine-doc DESIGN to have the changes applied and the addressed comments removed. The sidecar is gitignored; resolutions land via PR. See WORKFLOW.md → Doc review loop.
Q: Why is the design system in docs/DESIGN/ and not in the app's source?
A: It's the design deliverable — the contract. During Implement, frontend-lead imports or derives the app's real stylesheets from tokens.css / screen.css. Keeping the canonical system here keeps it reviewable as a doc and decoupled from any one app's build layout.
